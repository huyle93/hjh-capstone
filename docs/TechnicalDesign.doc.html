  1. Project Description

This project is being sponsored and proposed by the Liberty Mutual Information Technology Enterprise Technology Services (ETS) department. ETS is responsible for creating, designing and supporting enterprise solutions for all of Liberty Mutual departments. As part of that responsibility, ETS seeks to create a dataset for their Legal team to better understand key components in judicial outcomes.

An enhanced understanding of judicial outcomes will enable Liberty Mutual to choose the best representation for their insureds.

  1. Assumptions
  • All current documents are up to date.
  • PDF files provide complete and accurate information.
  • All the data/PDF files have fully public access and doesn’t require any login.
  • We assume that the data and resources we require to complete this project is available to us, or it will be made available to us.
  • We will also assume that there is no internal network infrastructure and that the users and clients are competent in their abilities to use a computer.

  1. Automated Data Retrieval API

One of the main way to collect data on the internet for this project is through scraping websites. Web scraping software access the World Wide Web directly using Hypertext Transfer Protocol. The process of web scraping can be controlled manually, or automated by collecting the unstructured data in a specific of time, or recursively. The Automated Data Retrieving Application Programming Interface (API) will receive an input from the user as a specific U.S State’s name. By the user’s given state’s name, the script will execute the correct State pre-coded version to retrieve data from that State.

The process started with finding the wanted data in type of pdf documents. Content in type of JSON or text are also considered. Next, the process will identify the request by range of the period of data and court division given from the user, for example: from 2014 to 2016, civil. The process will then classify the request and search for the web address which contains those specific information. When the software found the correct address which contains the wanted information, it will start searching for the PDF files and start the retrieval process to retrieve the wanted data into the given directory in user’s local machine. The web scraping process is developed using Python and Python libraries.

  1. Natural Language Processing

Natural Language Processing (NLP) is one of the most important part in this software. The NLP process includes 3 different steps.

  1. Extract raw data - the process of converting raw data from the previous process of data retrieving to feasible type of data like .txt (text format).
  2. Generate processed data - the process of data munging, which means cleaning and formatting the text content before using for analytics. This process is also known as preparing the data for applying data mining, machine learning algorithms.
  3. Search for important data - the process of teaching the computer to search for the important data from the processed data. This process will apply Natural Language Processing toolkits and libraries including the Natural Language Tool Kits (NLTK) and Google Cloud Language API’s Name Entity Recognition to extract wanted fields. The result from this step is to extract the key data from the lengthy processed data.

  1. Data Mining

Data mining is the process of applying data science algorithm or business rules to find the trends, relationship or building correlation models. In this process, firstly, the process will apply the business rules given from our client to seek for the relationship of data points. Next and potentially, the process will apply data mining algorithm Naive Bayes to find the insightful relationships and trends of our data points such as Attorney’s name.

Sample prediction using Naive Bayes: What is the probability of attorney John Zaikowski to win a case when the Judge is Thomas Durkin of Vermont Court

P(win | john zaikowski & thomas durkin)

= P(john zaikowski | win) P(thomas durkin | win) P(win)

= 3/19 * 4/19 * 12/19 = 0.1578 * 0.2105 * 0.6315 = 0.02097

Normalize John Zaikowski’s probability:

P(lose | john zaikowski & thomas durkin)

= P(john zaikowski | lose) P(thomas durkin | lose) P(lose)

= 2/19 * 2/19 * 7/19

= 0.1052 * 0.1052 * 0.3684

= 0.00407

Normalized rating for wins: 0.02097/(0.02097 + 0.00407) = 0.83 or 83%

Conclusion: Based on our data, John Zaikowski has 83% chance to win when Thomas Durkin is a given Judge.

  1. Data for Good

Data for good is the last step in our pipeline where the insightful data will be available to use. This process is to publish and give the access to developers or the end-users to access our database. There are 2 potential ways are RESTful GraphQL Application Programming Interface (API) and Web Application. The RESTful GraphQL API will respond to the request to access our databases with JSON file. The web application is the user friendly interface for non-technical users to apply business rules to query information from our database.

  1. Testing Strategy

We will be doing usability testing during the coupled stages of the project. For example, we will be showing our client once we have the data pull and organization. Once she agrees to the format and the looks of the data we have pull, then we will start storing it in the database. We will do the next test once we have successfully store all the data in the database(MongoDB). The final two testing will be done due the web application. We will be doing one in the mid way of creating an application to learn if users and clients is able to navigate around. The last and final test will be done in the beginning to mid way of  April once we have the application fully functionally and data search is working. After the user testing we will still have time left if there are any more change needed.
