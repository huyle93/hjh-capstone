  1. Project Description

This project is proposed and sponsored by the Liberty Mutual Information Technology Enterprise Technology Services (ETS). ETS is responsible for creating, designing, and supporting enterprise solutions for all of the Liberty Mutual. As part of that responsibility, ETS seeks to create a dataset for their Legal team to better understand key components in judicial outcomes. This project creates a software that aids Liberty Mutual’s Legal team in identifying the best attorney. The system mines data from publicly available websites, creating a source for further research and development to aid in this decision.

  1. Assumptions
  • All current documents are up to date.
  • PDF files provide complete and accurate information.
  • All the data/PDF files have full public access and doesn’t require any login.
  • We assume that the data and resources we require to complete this project is available to us, or it will be made available to us.
  • We will also assume that there is no internal network infrastructure and that the users and clients are competent in their abilities to use a computer.

  1. Automated Data Retrieval API

One of the main ways  to collect data on the internet for this project is through scraping websites. Web scraping software to access the  World Wide Web directly uses  Hypertext Transfer Protocol. The process of web scraping can be controlled manually, or automated by collecting the unstructured data in a specific time,  or recursively. The Automated Data Retrieving Application Programming Interface (API) will receive an input from the user as a specific U.S State’s name. By the user’s given state’s name, the script will execute the correct State pre-coded version to retrieve data from that State.

The process started with finding the wanted data in type of PDF  documents. Content in the type  of JSON or text is also considered . Next, the process will identify the request by the range  of the period of data and court division given by  the user, for example: from 2014 to 2016, civil. The process will then classify the request and search for the web address which contains those specific information.   When the software found the correct address which contains the wanted information, it will start searching for the PDF files and start the retrieval process to retrieve the wanted data into the given directory in the user’s local machine . The web scraping process is developed as a collection of object oriented Python classes.

There are three main classes are the retriver class, the urlgenerator class and the converter class. In this 1.0 version, we just scraped the court documents of State of Vermont. By having the object oriented design of classes with different attributes and methods, it sets the foundation for the system to add more court documents from other States.

  1. Natural Language Processing

Natural Language Processing (NLP) is one of the most important part of this software. The NLP process includes 3 different steps.

  1. Extract and explore data - the process of converting raw data (pdf format) from the previous process of data retrieving to feasible type of data like .txt (text format) then explore that unstructured dataset using NLP algorithms. Some of the algorithms and tools those we used to explore the dataset are: Gensim Word2vec, td-idf, pytextrank

 

Gensim Word2vec Outcome - finding similarity words

  1. Generate processed data - the process of data munging, which means cleaning and formatting the text content before using for analytics. This process is also known as preparing the data for applying data mining, machine learning algorithms.

Object Oriented Design of data processing procedure

  1. Search for important data - the process of teaching the computer to search for the important data from the processed data. This process will apply Natural Language Processing toolkits  and libraries, including the Natural Language Tool Kits (NLTK), Python regex, N-Gram and Google Cloud Language API’s Name Entity Recognition to extract wanted fields. The result of  this step is to extract the key data from the lengthy processed data.

  1. Data Mining

Data mining is the process of applying data science algorithms  or business rules to find the trends, relationship or building correlation models. In this process, firstly, the process will apply the business rules given from our client to seek in  the relationship of data points. Next and, potentially , the process will apply data mining algorithm Naive Bayes to find the insightful relationships and trends of our data points such as Attorney’s name.

Sample prediction using Naive Bayes:  What is the probability of attorney John Zaikowski to win a case when the Judge is Thomas Durkin of Vermont Court

P(win | john zaikowski & thomas durkin)

= P(john zaikowski | win) P(thomas durkin | win) P(win)

= 3/19 * 4/19 * 12/19 = 0.1578 * 0.2105 * 0.6315 = 0.02097

Normalize John Zaikowski’s probability:

P(lose | john zaikowski & thomas durkin)

= P(john zaikowski | lose) P(thomas durkin | lose) P(lose)

= 2/19 * 2/19 * 7/19

= 0.1052 * 0.1052 * 0.3684

= 0.00407

Normalized rating for wins: 0.02097/(0.02097 + 0.00407) = 0.83 or 83%

Conclusion: Based on our data, John Zaikowski has 83% chance to win when Thomas Durkin is a given Judge.

  1. Testing Strategy

We will be doing usability testing during the few stages of the project. For example, we will be showing our client once we have data pull and organize. Once she agrees to the format and the structure of the data we have pull, then we will start storing it in the database. We will do the next test once we have successfully stored  all the data in the database  ( MongoDB). The final two testing will be done during the web application. We will be doing one in the midway  of creating an application to learn if users and clients are  able to navigate around an understand the data. The last and final test will be done in the beginning to mid way in  April once we have the application fully functioning  and data search is working. After the user testing, we  will still have time left if there is  any more change needed. Along with the testing at specific area we will be showing the client every major step we have done in the project for her suggestion for improvement.
